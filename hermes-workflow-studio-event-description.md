# Hermes Workflow Studio Event Description

## Summary

Hermes Workflow Studio is a public workshop series on building governed AI workflows that teams can actually pilot.

The series uses one flagship workflow, `/account-action-brief`, to show the full Hermes lifecycle: build a source-grounded workflow with You.com, connect it to GTM tools, add governance and NemoClaw review gates, then package it as a reusable Hermes skill or playbook with a two-week pilot plan.

The goal is to make Hermes the hero. You.com provides live web intelligence, NemoClaw provides risk and approved-language review, and the rest of the GTM stack becomes context and action surfaces inside a governed Hermes workflow.

## Event Title

Hermes Workflow Studio: Build Governed AI Workflows with You.com

## Short Description

Build a reusable Hermes workflow that uses You.com for live web intelligence, connects to your GTM stack, routes risky claims through NemoClaw, and ends with a two-week production pilot plan.

## Event Description

Most AI demos stop at an answer. Real teams need workflows that can be reused, governed, reviewed, and piloted in production.

This public workshop series shows how to build a Hermes workflow from the ground up. We will start with a source-grounded account-action brief powered by You.com, connect it to CRM, docs, Slack, outreach, and review systems, add NemoClaw governance checks, then package the workflow as a reusable Hermes skill or playbook.

Across the series, attendees will see how Hermes supports the full workflow lifecycle: run, integrate, govern, reuse, and pilot.

The flagship example is `/account-action-brief`: a workflow that turns a target account into a source-backed brief, approved outreach angle, CRM-ready update, Slack review request, and two-week pilot plan.

## Four-Part Agenda

### Part 1: Build The First Hermes Workflow

Focus:
Turn a repeated AI task into a named Hermes workflow command with defined inputs, outputs, and source requirements.

What we will cover:
- Define the job: account research plus recommended next action.
- Create the `/account-action-brief` workflow shape.
- Use You.com Search for current structured results.
- Use You.com Contents to inspect key source pages.
- Use You.com Research when the question needs synthesis.
- Produce the first internal account-action brief.
- Convert the prompt into a reusable Hermes workflow card.

Attendee artifact:
First Hermes workflow card.

### Part 2: Connect The GTM Stack

Focus:
Show Hermes coordinating multiple tools while keeping the workflow understandable and reusable.

What we will cover:
- Map each integration to one job in the workflow.
- Add CRM context such as owner, stage, contacts, and next steps.
- Add approved messaging and playbooks from docs or knowledge bases.
- Add meeting, support, or product context where useful.
- Generate an internal brief and external-ready outreach draft.
- Prepare CRM, Slack/Teams, and outreach handoffs for human approval.

Attendee artifact:
Connector map with read/write mode, data used, data excluded, owner, and review requirement.

### Part 3: Govern The Workflow

Focus:
Let admins decide which users, tools, data, and actions are allowed inside a reusable Hermes workflow.

What we will cover:
- Define roles: runner, owner, reviewer, and admin.
- Define allowed tools and restricted tools.
- Define data that is off-limits.
- Add external-use review rules.
- Add NemoClaw checks for unsupported claims, competitive claims, privacy/security language, customer-sensitive details, and high-risk tool actions.
- Add approval states: internal-only, needs edit, approved for external use, and human review required.

Attendee artifact:
Governance checklist and review model.

### Part 4: Package And Pilot

Focus:
Turn the successful workflow into a reusable Hermes skill or playbook with owners, metrics, review rules, and rollout plan.

What we will cover:
- Finalize the workflow card.
- Define the output schema.
- Define pilot cohort and workflow owner.
- Define success metrics.
- Define failure handling and feedback collection.
- Build a two-week pilot plan.
- Map adjacent workflows: trusted answer product, support answer copilot, competitive intelligence brief, renewal prep, and partner brief.

Attendee artifact:
Pilot launch kit with workflow card, connector map, NemoClaw governance checklist, quality scorecard, two-week pilot plan, and expansion backlog.
